# Docs Module Guide

This directory holds repository-facing documentation.

Working rules:

- Keep each document focused on one workflow or one question.
- Prefer high-level workflow explanations over file-by-file narration.
- When documenting an algorithm, include the prompt sequence, the runtime settings, the factorial factors, the artifact outputs, and the verification path.
- Use ASCII diagrams when a state transition or branching flow is easier to understand visually.
- If a document becomes broad, split it into a focused companion document instead of expanding it indefinitely.
