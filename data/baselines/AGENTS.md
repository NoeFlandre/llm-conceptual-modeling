# Baselines Guide

This directory contains committed baseline outputs for comparison workflows.

Purpose:

- Preserve deterministic baseline corpora used by analysis and verification.
- Keep baselines separate from imported results and ad hoc debug artifacts.

Working rules:

- Treat baseline files as reference outputs.
- Keep algorithm-specific baseline subtrees stable.
- Add a deeper guide when a baseline subtree has its own file naming or layout rules.
- Do not mix baseline outputs with generated analysis artifacts.
