# Revision Tracker Guide

This directory contains reproducible analysis outputs for revision-tracking work.

Purpose:

- Preserve dated summary outputs used to support manuscript revision and tracker findings.
- Keep these outputs separate from live debug artifacts.

Working rules:

- Keep each dated run in its own subdirectory.
- Add a deeper guide when a dated run has its own internal layout.
- Treat the tracked CSVs as derived evidence tied to a specific revision date.
- Do not mix exploratory debug outputs into this subtree.
