# Post Revision Debug Artifacts Guide

This directory contains live-provider debugging and replay artifacts created after the freeze.

Purpose:

- Keep post-freeze exploratory outputs separate from imported historical results.
- Document the canonical debug-artifact layout used by revision work.

Working rules:

- Follow the layout described in the local `README.md`.
- Keep exploratory runs and canonical audited matrices separated by subdirectory.
- Do not write revision-tracker outputs here.
- Treat the artifacts as append-only evidence for debugging and review.
